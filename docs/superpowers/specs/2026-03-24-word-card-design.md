# 词卡网站设计规格

## 背景

目标是把 `ljg-word -> ljg-word-flow -> ljg-card -i` 的方法迁移到一个本地可运行的网站里：

1. 用户输入一个中文词或英文单词
2. 服务端调用 LLM 先完成“解词”
3. 前端把解词结果渲染成信息图式词卡
4. 用户下载 PNG

这里借鉴的是方法论，而不是原命令行交互形式。最关键的约束仍然保持不变：先解词，后铸卡；卡片内容来自解词结果，而不是普通字典释义。

## 产品边界

### 输入规则

- 英文输入：单个英文单词
- 中文输入：1 到 4 个汉字
- 不支持英文短语
- 不支持中英混输
- 不支持批量并行输入

### 输出规则

每次只生成一张词卡，内容结构沿用 `ljg-word`：

1. 标题行
2. 原始画面
3. 核心意象公式
4. 深层解释
5. 一语道破

### 非目标

- 不做登录
- 不做数据库持久化
- 不做历史记录
- 不做多词并行
- 不做静态词库兜底

## 技术决策

### 技术栈

- 前端：Vite + Vanilla JavaScript + CSS
- 服务端：Vite dev middleware 提供 `/api/analyze`
- 图片导出：html2canvas
- LLM 提供方：OpenRouter，配置从 `.env` 读取

### 决策理由

- 保持现有项目结构，避免为单页产品引入额外框架
- 需求集中在体验与提示词质量，不在复杂状态管理
- 现有实现已具备基本视图切换和导出能力，适合增量完善

## 视觉方向

词卡不做通用 SaaS 风格，而是做“编辑性纸本信息图”方向：

- 首页像一张静置的封面页，突出“词有纹理”的感觉
- 词卡主体像一张经过编排的阅读海报，而不是 App 卡片
- 使用高对比衬线字体、等宽标记与大留白
- 用低饱和底色、轻颗粒、细线分隔与有限强调色，保持 `ljg-card` 的内容优先气质

## 数据结构

服务端统一返回：

```json
{
  "word": "string",
  "inputType": "english|chinese",
  "phonetic": "string",
  "translation": "string",
  "originalImage": "string",
  "coreSymbolParts": ["string", "string", "string"],
  "coreSymbolResult": "string",
  "explanation": "string",
  "epiphanyEn": "string",
  "epiphanyCn": "string",
  "mood": "contemplative|sharp|warm|technical|research|creative|business"
}
```

其中：

- `explanation` 仍使用 `||` 作为段落分隔符，方便前端直接渲染
- `inputType` 替代当前的 `isChinese` 布尔值，让前后端语义更清晰
- `mood` 决定整张词卡的配色倾向

## 解词策略

### 英文

严格沿用 `ljg-word` 的核心要求：

- 不是翻译任务
- 要交代词源对应的原始物理画面
- 要把核心语义压成公式
- 要解释词源、现代语义与跨领域用法的内在联系
- 最后给出有哲学高度的双语金句

### 中文

中文是本项目扩展，不是原 skill 原生能力，但解构方式保持一致：

- 若是单字，优先回到字形、构件、古义与意象源头
- 若是双字及以上词语，优先解释词组内部张力、组合关系与语义重心
- 依然要产出“原始画面 + 核心意象公式 + 深层解释 + 金句”
- 解释应偏洞见，不应退化为词典释义堆砌

## 页面结构

### 首页

- 品牌标题
- 一句话副标题
- 输入框
- 规范提示
- 推荐示例词

### 词卡页

- 顶部导航：返回、重新生成、下载 PNG
- 卡片头部：编号、词本体、音标或拼音、简译
- 卡片主体：
  - 原始画面
  - 核心意象公式
  - 深层解释
  - 一语道破
- 辅助信息：
  - 输入类型标签
  - 生成日期

## 错误处理

- 输入为空：直接提示
- 输入类型不合法：给出明确规则
- `.env` 缺失：服务端返回配置错误
- LLM 调用失败：返回用户可理解的错误文案
- LLM 返回非 JSON：服务端尝试清洗；仍失败则返回解析失败

## 测试与验证

至少覆盖以下行为：

1. 输入校验函数
2. LLM 响应解析函数
3. 中文与英文提示词分流
4. 前端渲染结构不因缺项崩溃
5. 构建成功
6. 本地 `localhost` 可启动
