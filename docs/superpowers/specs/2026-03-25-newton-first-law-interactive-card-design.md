# 牛顿第一定律互动概念卡设计说明

## 目标

在现有“词卡”网站中落地第一张互动概念卡样板，主题为“牛顿第一定律”。这张卡不是普通说明页，也不是完整物理模拟器，而是把 `Plain + Learn + 互动模拟` 压成同一张会动的概念卡。

目标用户是 12 岁以上、对物理有兴趣但并不一定具备系统训练的用户。产品目标不是覆盖全部物理定律，而是验证一种新的互动知识产品形态是否成立。

## 产品定义

这是一张“未来实验舱感”的互动概念卡，核心体验是：

1. 先被一句人话带入概念
2. 立即进入 3D 轨道小车实验场景
3. 通过拖动少量关键变量观察现象变化
4. 在实时解释中完成误解纠偏
5. 最后用正式定义和一句顿悟收束理解

它应当同时满足三个条件：

- 物理表现不违背第一定律的核心逻辑
- 交互足够有吸引力，用户愿意反复试参数
- 信息结构足够克制，仍然保留“卡片感”而不是变成实验平台

## 第一版范围

### 包含

- 单一概念：牛顿第一定律
- 单一主场景：轨道上的小车
- 三个核心变量：
  - 摩擦系数
  - 初速度
  - 外力模式（无外力 / 短推一下 / 持续推）
- 一种观察重点切换（更关注位移 / 速度与受力）
- 实时解释区
- 正式定义区
- 误解纠偏区
- 顿悟句区
- 可导出 PNG 的概念卡静态视图

### 不包含

- 通用概念卡模板系统
- 多概念切换
- 完整实验平台
- 高级刚体碰撞、多对象相互作用
- 用户历史记录、账户系统、分享系统

## 认知目标

用户离开页面前应至少获得以下理解：

1. 小车停下不是因为“运动自己会消失”
2. 当合外力为零时，物体保持静止或匀速直线运动
3. 摩擦是现实世界里最常见的“额外改变因素”
4. 第一定律在本质上讨论的是“改变运动状态为什么需要原因”

## 体验主线

### 1. 一句人话

页面首屏先呈现一句极简的人话解释：

> 不推它，它不会自己改主意。

这句话必须能在用户看见公式前先建立直觉。

### 2. 主互动区

用户进入页面后，主视线应立即落到 3D 场景。小车和轨道是视觉中心，控制台紧贴主场景，形成“实验台”感。

主互动区必须满足：

- 小车自动运动或可被立即启动
- 变量变化后有即时视觉反馈
- 不出现“隐形边界导致速度被凭空清零”的违背直觉现象

### 3. 实时解释

实时解释根据当前状态切换文案，而不是固定一段说明。文案策略如下：

- `摩擦接近 0 + 无外力`
  - 强调“接近匀速直线运动”
- `摩擦较高 + 无外力`
  - 强调“减速来自摩擦，不是运动自然消失”
- `持续推`
  - 强调“当前已不属于第一定律最核心的零合外力情形”
- `短推 + 无摩擦`
  - 强调“短推结束后仍继续前进”

### 4. 正式收束

页面下半段用更克制的版式呈现：

- 正式定义
- 误解纠偏
- 顿悟句

其作用是把用户刚刚在互动区“看懂”的内容压缩成可记忆语言。

## 视觉方向

采用“未来实验舱 + 科学展品收边”的混合风格：

- 3D 区强调沉浸与装置感
- 文本区强调阅读性与克制
- 避免赛博朋克噪音和游戏 UI 堆叠

视觉关键词：

- 冷静、精密、受控
- 有材质变化
- 有速度与力的可视反馈
- 有轻微景深 / 阴影 / 拖影
- 文本部分保持高质量知识展品气质

## 技术方向

前端技术栈升级为：

- React
- React Three Fiber
- Drei
- Rapier
- 保留 Vite 作为构建与本地开发工具

技术分工如下：

- React：整页结构、控件、解释文案、页面状态
- R3F：3D 轨道场景渲染
- Drei：常用相机 / 控件 / 帮助组件
- Rapier：物理状态与运动更新

## 模块边界

建议拆分为以下模块：

- `src/app/*`
  - 页面壳与路由入口
- `src/features/newton-first-law/*`
  - 牛顿第一定律页面、状态、解释逻辑
- `src/features/newton-first-law/scene/*`
  - 轨道、小车、灯光、相机等场景对象
- `src/features/newton-first-law/copy/*`
  - 实时解释模板、正式定义、误解纠偏、顿悟句
- `src/lib/*`
  - 通用工具与现有导出/分析逻辑

## 风险与约束

### 1. 当前项目是 Vanilla JS

升级到 React 需要重构前端入口，但后端 `/api/analyze` 可以继续复用。

### 2. 第一版不能把模拟做成“玩具物理”

哪怕是样板，也不能出现明显违反牛顿第一定律的默认行为。

### 3. PNG 导出不适合完整导出“运行中三维画面”

第一版的 PNG 更适合导出“静态状态下的概念卡快照”，而不是录制动画。

## 成功标准

第一版达成以下标准即视为成功：

1. 用户拖动参数时愿意持续尝试
2. `摩擦 = 0 + 短推一下` 时，小车不会凭空停下
3. 用户能从实时解释中获得明确纠偏
4. 页面仍保留概念卡的阅读节奏，而不是沦为纯模拟器
